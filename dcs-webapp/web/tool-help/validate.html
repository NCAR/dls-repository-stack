Tool Help - Validate

The Validate button checkes the input values for the current page against the acceptable values as defined in the XML Schema.

Currently, SchemEdit does not require validation either when moving from page to page or before writing to disk. The policy of leaving validation up to the user can be easily changed, however, if required by a particular workflow.

Implementation Details

SchemEdit uses the Sun XML Datatypes Library to validate field values. The Sun Datatype Library contains validators for almost all of the datatypes defined in the XML Schema Part 2 specification. Schemas define data types by extending these built-in datatypes. SchemEdit automatically creates validators for derived data types.

There are a few known problems with the Sun Datatype library, including:

  • language type - which accepts nearly any string, and
  • some date/time related types which do not work properly because the XML Schema Part 2 is broken with respect to dates (for details, see [1] [2] [3] and [4] )
These limitations will be addressed by creating alternate validation classes for the affected types.
