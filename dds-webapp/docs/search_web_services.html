Description

The DLESE Search Web Service is used by internal and external developers to search the library holdings and access metadata from DLESE's library repository. The service is designed to provide rapid, distributed search and retrieval for items in the library. Developers may use the service to create a variety of custom search interfaces, access the data in the DLESE repository, or conduct research about the library repository and it's contents. A developers toolkit and template is provided to aid in the rapid development of new search interfaces and as a tool for learning the service protocol. A number of search interfaces and systems have been created using the service such as the Florida Center for Ocean Science Excellence (FCOSEE) search page, the News and Opportunities portion of the DLESE web site, and the Contributor Acknowledgement System. The service is formally described in the service documentation and specification document.

The service supports natural language textual searching over metadata and content, searching by date and field, sorting by date and field, searching by geospatial bounding box coordinates, discovery of the controlled vocabularies that are part of the library including grade ranges, subjects, resources types and content standards, checking for the existence of a URL, and other functionality. A full range of Information Retrieval features are exposed through the service such as term, field and phrase searches, term and term/field boosting, term stemming, wildcard and fuzzy searches, term proximity searches, and other functionality. The DLESE Search Web Service protocol is an example of a Representation State Transfer (REST) style architecture.

Audience

Library developers, web designers, educational designers, software engineers, and researchers.

How this component is used

The search web service is used to develop discovery interfaces for educational resources, news and opportunities, collections and other content and metadata associated with the library. The service is used by both internally by developers local to the DPC as well as external developers.

Functional requirements

  • Support distributed access to DLESE's search functionality and metadata repository.
  • Support the design, development and deployment of custom interfaces that access and display DLESE resources.
  • Use technology that is readily available, relatively easy use, and can be developed using standard open source tools.

Technologies used

The service is implemented using Java 1.4, J2EE, Struts, Lucene 1.4, Tomcat 5, Dom4j, JSTL 1.1 tag libraries. Clients that access the service are implemented using application software such as Tomcat / JSP, PHP, Java or other environments that are able to communicate using web service protocols and parse XML data for display or data analysis purposes.
